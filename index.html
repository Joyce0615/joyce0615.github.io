🚀 Featured Project

Solopreneur Team

Multi-Agent AI Platform for Creators & Founders

A platform where 5 specialized AI agents — Marketer, Designer, Bookkeeper, Customer Support, and Sales — collaborate to run a solopreneur's business. Built solo from scratch: 88 source files, ~10,500 LOC. The technical depth is in the agent harness — not the model, but the system that makes agents reliable, context-efficient, and safe.

▸

Cache-aware prompt assembly — system prompts split at a dynamic boundary; static sections (~4K tokens) cached per request, reducing latency and inference cost

▸

4-stage context compression pipeline — snip → micro-summarize → collapse → drop; activates at 80% token budget to prevent agent amnesia mid-conversation

▸

Fail-closed tool governance — every tool has a requiresApproval: true default, Zod schema validation, agent-type whitelist, and structured ToolResult

▸

Multi-agent memory scoping — per-agent memory isolation with cross-agent shared project context; prevents one agent's state from contaminating another

▸

Multi-provider LLM abstraction — unified interface across OpenAI, Anthropic, and Gemini via Vercel AI SDK v6; swap providers per agent without code changes

📔 Featured Project

AI Diary Companion

LLM Character System with Memory & Prompt Engineering

A full-stack AI journal where users choose a character companion (therapist, philosopher, coach, artist) for daily conversations. The character listens in-persona, then composes a diary entry in the user's own voice. 38 source files, ~3,500 LOC. The challenge was making characters feel real — building the prompt architecture and memory system that creates genuine continuity.

▸

Layered prompt architecture — static (character identity, norms, safety) + dynamic (memories, diary summaries, date/time) split for cache efficiency across every request

▸

Character behavior codification — characters defined as 3-layer behavioral specs (identity → personality → voice style), not descriptions; prevents model drift over long sessions

▸

Memory lifecycle management — incremental extraction after each chat + weekly consolidation; importance-scored with natural-language timestamps, injected as contextual hints

▸

Structured output parsing — diary generation prompt instructs model to return JSON with title, narrative, mood, tags; parsed and validated before DB write

▸

Conversation windowing — last 20 messages in context window; older messages in DB but excluded; prevents token starvation on long-running sessions

▸

Character skill system — per-character skill definitions with keyword, mood, and intent triggers; detectActiveSkills() injects specialized behavior instructions as a second prompt section at runtime

🏆 Featured Project

Kaggle Team

Multi-Agent System for Kaggle Competition Optimization

A CLI-driven multi-agent team — Researcher, PM, and Engineer — that collaborates through a 4-phase evolution loop to progressively improve Kaggle competition scores. 75 source files, ~6,300 LOC. Built around a score-informed feedback loop: every agent sees submission history, CV-LB gap trends, and community signals before acting.

▸

4-phase evolution loop — Research → Strategy → Build → QA; each round's scores, gaps, and community findings feed directly into the next round's agent context

▸

Living community intelligence — incremental scanning of Kaggle notebooks and discussions each round; agents see only the delta (new content since last scan)

▸

BUILD_ON vs REBUILD decision — PM explicitly chooses whether to iterate on the previous round's solution or start fresh, citing score trends and community evidence

▸

Cost & turn tracking — per-phase token cost and turn count extracted from stream events; persisted in state.json with a summary table printed after each round

▸

Round-table retrospectives — periodic cross-round review session (configurable interval) synthesizing score trajectory, bottlenecks, and strategic pivot recommendations
